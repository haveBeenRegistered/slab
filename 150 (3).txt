64.Java EEの3層構造とは何ですか。
各層（レイヤー）の具体的な役割
① プレゼンテーション層（画面・制御）
役割：ユーザー（ブラウザ）との窓口です。画面を表示したり、ユーザーが入力したデータを受け取って「ビジネスロジック層」へ引き渡します。

Java EEでの主な技術：JSF (Jakarta Server Faces)、JSP、Servlet、JAX-RS（RESTful APIのコントローラー）


② ビジネスロジック層（業務処理）
役割：システムの「頭脳」となる部分です。ビジネス上のルールや計算、処理手順（ロジック）をここに集約します。この層は画面のデザインや、データベースの種類（MySQLやOracleなど）を一切気にしません。

Java EEでの主な技術：EJB (Enterprise JavaBeans)、CDI (Contexts and Dependency Injection)


③ データアクセス層（永続化・DB操作）
役割：データベースなどの外部システムとデータをやり取りする専門の層です。ビジネスロジック層からの指示を受けて、データの登録（Insert）や検索（Select）を行います。

Java EEでの主な技術：JPA (Jakarta Persistence / 旧Java Persistence API)、JDBC
